# Пополняемые вклады + интеграция вкладов в список счетов

Дата: 2026-07-03 · Версия: v1.20.0

## Задача

1. Пополняемый вклад: пополнение суммой на конкретную дату.
2. График «Рост накоплений» (+вклады) учитывает пополнения корректно: до даты
   пополнения — стоимость без него, после — с ним (проценты на пополнение
   набегают с его даты).
3. Кнопка «Мои вклады» — выше «Целей накоплений», под «Текущими счетами».
4. Вклады видны в списке «Текущие счета» отдельной группой с пометкой «вклад»
   (по аналогии с пометкой «кредит»). Управление остаётся на странице вкладов.

## Модель данных

`DB.deposits[i].contributions = [{id, date: 'YYYY-MM-DD', amount}]` —
опциональное поле, отсутствует у старых вкладов (backward compatible).
Синхронизируется внутри записи вклада (LWW по `updatedAt` всего вклада —
любое изменение пополнений штампует `updatedAt` вклада).

## Расчёт `depositValueAt(d, dateStr)`

Учитываются только пополнения с `date <= min(dateStr, endDate)`:

- **Капитализация monthly**: `amount·(1+r/12)^(days(open→stop)/30.4375) +
  Σ c.amount·(1+r/12)^(days(c.date→stop)/30.4375)`.
- **% в конце ('end')**, до endDate: тело + пополнения (без процентов).
  На/после endDate: `finalAmount`, если задан; иначе простые проценты на тело
  от openDate + на каждое пополнение от его даты.
- **finalAmount** задан + monthly: на/после endDate — ровно `finalAmount`.

## Производная ставка (`_calcDepRate`)

С пополнениями закрытой формулы нет → бисекция по ставке (0–300%, ~60
итераций) до совпадения расчётной стоимости на endDate с `finalAmount`.
Без пополнений — прежние закрытые формулы. При добавлении/удалении
пополнения у вклада с `finalAmount` ставка пересчитывается.

## UI

- **Карточка вклада** (страница «Вклады»): кнопка «+ Пополнить» (для
  незакрытых) → модалка (дата = сегодня, сумма). Строка суммы показывает
  общее тело (amount + пополнения) → на закрытии; при наличии пополнений —
  подстрока «тело X + пополнения Y».
- **Модалка редактирования вклада**: список пополнений с удалением (×).
  Удаление — сразу (saveDB + updatedAt), не по «Сохранить».
- **Прогноз «Сколько будет к дате»**: тело = amount + пополнения с датой
  ≤ даты прогноза.
- **Список «Текущие счета»**: после кредитных банков — строки живых вкладов:
  имя + бейдж «вклад» (синий), сумма = `depositValueAt(d, today())`,
  подстрока «до <дата> · N%» или «✓ срок вышел». Клик → страница вкладов.
  В «Всего активов» вклады НЕ входят (как раньше — строка «с вкладами»).
- **Кнопка «Мои вклады»**: между списком счетов и «Целями накоплений».
  «Прогноз накоплений» остаётся внизу.

## Не делаем (YAGNI)

- Списание пополнения с банковского счёта (пользователь сам обновляет банк).
- Перенос вкладов в менеджер банков / история записей по вкладу.
- Синхронизация пополнений отдельными записями (гранулярность — весь вклад).
